## Key Discoveries

### Session Start Pattern
- **What I thought:** "Before X, read Y" = check INDEX.md before every task
- **What it means:** "ON SESSION START" = read INDEX.md + memory banks ONCE at start
- **Impact:** Don't check INDEX.md every time, just once per session

### Memory System Intended Usage
- Load memory once at session start (INDEX.md + relevant banks)
- Use throughout session
- Check INDEX.md only when uncertain what to read
- "FOR X" sections = task-type-specific guidance (long tasks, troubleshooting)

### Agent Timeout Setting
- Default: 600 seconds (10 minutes)
- Found: Can be increased via `agents.defaults.timeout` in config
- Changed to: 1800 seconds (30 minutes)
- No restart needed (applies next session)

### Sub-Agents Capability
- Configured: Max 4 concurrent sub-agents
- Can use for: Parallel tasks, long-running background work, specialized expertise
